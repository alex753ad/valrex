# 🔍 Crypto Screener → Telegram

Скрипт сканирует все USDT-фьючерсы на Binance и отправляет уведомления в Telegram-канал,
когда монета показывает высокую волатильность (NATR) и объём.

---

## 📦 Установка

```bash
pip install aiohttp
```

---

## ⚙️ Настройка (файл screener.py)

Откройте `screener.py` и заполните блок НАСТРОЙКИ в начале файла:

| Параметр | Описание |
|---|---|
| `TELEGRAM_BOT_TOKEN` | Токен бота от [@BotFather](https://t.me/BotFather) |
| `TELEGRAM_CHANNEL_ID` | `@username` канала или числовой ID (`-1001234567890`) |
| `NATR_MIN` | Минимальный NATR (%) для сигнала (по умолчанию `0.5`) |
| `VOLUME_5M_MIN` | Минимальный объём за 5 мин в USDT (по умолчанию `100 000`) |
| `SCAN_INTERVAL_SEC` | Интервал между сканированиями (по умолчанию `60` сек) |

---

## 🤖 Создание бота и добавление в канал

1. Напишите [@BotFather](https://t.me/BotFather) → `/newbot` → получите токен
2. Добавьте бота в ваш канал как **администратора** с правом публикации
3. Вставьте токен и ID канала в `screener.py`

---

## ▶️ Запуск

```bash
python screener.py
```

## 🔧 Как работает

1. Каждые N секунд загружает список всех USDT-фьючерсов с Binance Futures
2. Для каждой монеты получает 1-минутные свечи → считает ATR за 14 периодов → NATR (%)
3. Получает 5-минутные свечи → берёт объём последней завершённой свечи
4. Если NATR ≥ порога И объём ≥ порога → отправляет сообщение в Telegram
5. Дедупликация: одна монета не отправляется чаще раза в 5 минут

---

## 🛡️ Защита от лимитов

- Символы сканируются параллельно пачками по 20
- Пауза 0.3 сек между пачками
- Кэш отправленных сигналов предотвращает спам
